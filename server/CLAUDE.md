# server (@devdigest/api) — module map

Fastify API: imports repos/PRs, indexes a repo via repo-intel, stores agents,
and runs the reviewer (diff → reviewer-core → grounded findings). Read this
first; depth is in `server/README.md`.

## Local commands
- `pnpm dev` (:3001) · `pnpm db:migrate` · `pnpm db:seed` (idempotent) · `pnpm typecheck`
- unit (hermetic): `pnpm exec vitest run --exclude '**/*.it.test.ts'`
- integration (real PG): `pnpm exec vitest run .it.test` · both: `pnpm test`

## Conventions (non-default)
- A feature = `modules/<name>/{routes,service,repository}.ts`, registered with ONE
  import + ONE entry in `modules/index.ts`. Registration is static, not filesystem autoload.
- Routes declare Zod `params`/`body` schemas via `fastify-type-provider-zod` — the same
  schema validates the request AND serializes the response. Don't hand-roll `Schema.parse`.
- All external deps go behind a port and are resolved through the DI container
  (`platform/container.ts`). Services depend on the interface; tests inject mocks
  (`adapters/mocks.ts`) via `ContainerOverrides`. Don't `new` an adapter inside a service.
- Shared cross-cutting repos (agents, reviews) hang off the container
  (`container.agentsRepo`), not reached into from another module's folder.
- A DB-backed test (imports `test/helpers/pg.ts`) MUST use the `*.it.test.ts` suffix,
  or the unit/integration split breaks.

## Gotchas
- Migrations are NOT applied on boot — run `pnpm db:migrate` (pgvector via migration 0000).
- On boot the app reaps orphaned `running` agent_runs. This assumes ONE API instance
  per DB; replicas would wrongly reap each other's live runs.
- The review pipeline itself is pure and lives in `reviewer-core`. This module owns only
  I/O: repo-intel context resolution + persistence + SSE — see `modules/reviews/run-executor.ts`.
- repo-intel degrades silently: an unindexed repo yields an empty repo map and the prompt
  falls back to diff-only. Don't treat empty enrichment as an error.

## Do NOT touch
- `src/db/migrations/*` — generated by Drizzle Kit (`pnpm db:generate`); don't hand-edit.
- `src/vendor/shared/*` — vendored `@devdigest/shared`; changing it ripples to every package.
